BBS+ Signatures

IETF Draft

Zero-knowledge proofs from multi-message signatures. Unlinkable presentations. Privacy gold standard for selective disclosure. Conceptual JSON shown below.

Key Pair BLS12-381 
{
  "publicKey": { "curve": "BLS12-381", "group": "G2", "value": "<compressed-g2-point>" },
  "secretKey": { "type": "scalar", "value": "<256-bit-scalar>" }
}
Signature Input & Output Conceptual 
// Input — each message becomes a separately-disclosable claim
{
  "header": "<context-bytes>",
  "messages": [
    "given_name:Alice",
    "family_name:Smith",
    "dateOfBirth:1990-06-15",
    "age_over_18:true",
    "document_number:DL-123456789"
  ]
}

// Signature
{ "A": "<compressed-g1-point>", "e": "<scalar>" }
Selective Disclosure Proof ZKP 
{
  "disclosed_indexes": [0, 3],
  "disclosed_messages": ["given_name:Alice", "age_over_18:true"],
  "proof": {
    "A_bar": "<blinded-g1-point>",
    "commitments": ["<scalar>", "<scalar>", "<scalar>"],
    "challenge": "<scalar>",
    "responses": ["<scalar>", "<scalar>", "<scalar>"]
  },
  "nonce": "verifier-supplied-challenge"
}
Key property: Two presentations from the same credential are cryptographically unlinkable. Pairing-based on BLS12-381. Each message maps to its own generator point.
